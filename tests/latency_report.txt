  ARCHITECTURE NOTE:
  ─────────────────
  The strategy_offline node internally calls build_strategy_context(),
  which re-invokes tire_agent, weather_agent, and RAG sequentially.
  This means data is fetched TWICE in the LangGraph flow:
    1st: by the standalone tire/weather/rag nodes (parallel in graph)
    2nd: by strategy_node → build_strategy_context() (sequential)

  FIX: Modify strategy_node to read pre-computed results from state
  instead of re-running build_strategy_context(). This would halve
  the data-fetching time. FastF1 caching mitigates this for cached
  sessions, but the first run for a new session would benefit greatly.
