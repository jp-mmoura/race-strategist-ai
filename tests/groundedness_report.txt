======================================================================
  SUMMARY & DIAGNOSIS
======================================================================

  KEY FINDING: The offline strategy generator does NOT use RAG data.
  The pipeline retrieves historical context via ChromaDB + FastF1,
  but the offline path reads ONLY from the tire_agent and weather_agent.
  The RAG context_text is appended to the prompt for LLM consumption only.

  CONSEQUENCE: In offline mode, the strategy is fully 'grounded' in
  tire data (it literally uses the winner's actual stints), but the
  RAG retrieval is wasted work. In LLM mode, groundedness depends on
  whether the LLM faithfully follows the data in its prompt.

  RECOMMENDATIONS:
  1. For offline mode: no hallucination risk (rule-based), but consider
     using RAG data to cross-validate compound recommendations.
  2. For LLM mode: add a post-generation check that extracts compounds
     from the LLM response and validates against tire_agent data.
  3. The retriever is working correctly — it returns relevant data.
     The issue is on the CONSUMER side, not the RETRIEVER side.
